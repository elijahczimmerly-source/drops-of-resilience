# PR Noise Correlation Length Sweep

## Goal

Find the `correlation_length` value for PR that minimizes RMSE without regressing Ext99 or WDF. The current value (35.0 px) was set heuristically. This sweep tests whether a different value improves RMSE.

## Background

The stochastic noise field is generated by convolving white noise with a Gaussian kernel of width `correlation_length` pixels. This controls the spatial scale of the noise patterns:
- **Too short** (e.g. 10 px): noise is pixel-scale speckle, adding variance at scales smaller than real storm structure. Pure excess variance = wasted RMSE.
- **Too long** (e.g. 100 px): noise is nearly uniform across the domain, can't create realistic sub-GCM storm heterogeneity.
- **Just right**: noise creates spatial patterns at the scale of actual precipitation variability, minimizing wasted variance.

The noise adds ~0.7 to RMSE (9.91 stochastic vs 9.21 deterministic). If even a fraction of that 0.7 is due to wrong-scale noise, correcting the correlation length could reduce RMSE meaningfully.

**IMPORTANT:** This sweep changes `correlation_length` for PR **only**. Other variables keep their current values (tasmax/rsds: 100, wind: 50, tasmin/huss: 35). The correlation length is set inside `_process_variable()` in `test8_v2_pr_intensity.py` — see the code change section below.

- **Noise debias is OFF.** `DOR_MULTIPLICATIVE_NOISE_DEBIAS=0` for all runs. Do not change it.
- **Baseline run:** `experiment_wdf_factor_1p65_216` (216x192, blend 0.65, debias off, seed 42, WDF factor 1.65). Use that run's Table1 CSV as the reference for all metric comparisons. If that run's Table1 is not available locally (it was output to `D:\WRC_DOR_runs\`), use `experiment_wdf_factor_1p50_216` as a close proxy — the only difference is WDF/Bias (RMSE differs by 0.001).

If tight, add `TEST8_SKIP_NPZ_SAVE=1` if that env var exists, or be prepared for `No space left on device` errors (Table1 CSVs are written before NPZs, so metrics will still be available even if NPZ save fails).

Change **only** the PR case to read from an env var. Replace the `elif var_name in ["tasmin", "huss", "pr"]:` block with:

    if _cl:

        corr_len = 35.0  # Mesoscale moisture/storm structures

    corr_len = 35.0  # Mesoscale moisture/storm/temp structures

This is the **only code change**. Do not change anything else in the pipeline. Do not touch the noise factor, the WDF threshold, the blend, or any other parameter.

## Sweep values

Sweep `DOR_PR_CORR_LENGTH` over: **{15, 25, 35, 45, 55, 70}**

- 35 is the current default (control — should exactly reproduce `experiment_wdf_factor_1p50_216` metrics).
- 15 and 25 test shorter (more localized storm patterns).
- 45, 55, 70 test longer (broader, smoother noise).

### Interpreting results

- **If a corr_len value beats RMSE while meeting all criteria:** That's the new default. Update line ~935 in `test8_v2_pr_intensity.py`.
- **If corr_len=35 is already optimal (or very close):** The correlation length is not the problem. Close this task. Pivot to the next idea in `BRAINSTORMING.md` — but read the instructions there first (ask Elijah for a Claude Code plan before implementing).
- **If shorter corr_len helps RMSE but hurts Ext99:** There's a tradeoff. Record the results and flag for discussion — do not change the default without approval.
- **If the control run (corr_len=35) does not exactly match `experiment_wdf_factor_1p50_216` metrics:** Something is wrong. Stop and investigate before interpreting the sweep.
